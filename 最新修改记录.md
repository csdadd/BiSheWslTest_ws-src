# 修改记录

## 问题描述
为日志系统添加一个新的日志级别 `COLLISION`，用于区分碰撞检测日志和普通 ERROR 日志，并添加配套的实时日志、历史日志的筛选显示功能。

## 修改文件列表
- `renwu/loglevel.h`: 添加 COLLISION 日志级别枚举值
- `renwu/logutils.h`: 添加 COLLISION 级别的字符串转换函数
- `renwu/systemmonitorthread.cpp`: 修改碰撞检测使用 COLLISION 级别
- `renwu/mainwindow.ui`: 添加实时日志和历史日志的 COLLISION 筛选复选框
- `renwu/mainwindow.cpp`: 更新筛选逻辑支持 COLLISION 级别

## 解决方案摘要
- **问题分析**：碰撞检测日志使用 `LogLevel::ERROR` 级别，与其他错误日志混合，不便于独立筛选查看
- **解决思路**：添加专用 `COLLISION` 日志级别，修改碰撞检测使用新级别，并在 UI 中添加对应的筛选复选框
- **关键修改**：
  1. 在 `loglevel.h` 枚举中添加 `COLLISION = 7`
  2. 在 `logutils.h` 中添加字符串转换支持
  3. 在 `systemmonitorthread.cpp` 中将碰撞日志级别从 `ERROR` 改为 `COLLISION`
  4. 在 `mainwindow.ui` 中添加两个 COLLISION 复选框
  5. 在 `mainwindow.cpp` 中更新 7 个筛选相关函数
- **验证方法**：
  ```bash
  cd /home/w/wheeltec_ros2
  colcon build --packages-select renwu
  source install/setup.bash
  ros2 run renwu renwu
  # 验证实时日志和历史日志区域显示 COLLISION 复选框并可正常筛选
  ```
